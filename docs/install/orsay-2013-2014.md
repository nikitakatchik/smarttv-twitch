# Install Twellie on an Orsay TV (2013–2014, F & H series)

For Samsung Smart TVs from **2013 (F)** and **2014 (H)** — the 5th character of
the model number is the year letter, e.g. `UN55`**`F`**`8000` (2013) or
`UN55`**`H`**`6350` (2014).

> Twellie is an **unofficial** viewer for Twitch. Not affiliated with Twitch,
> Samsung, or Tizen.

Orsay TVs have **no “Developer mode” toggle** like Tizen — that `12345` method is
2015+ only. Instead you sign Smart Hub into a built-in account named **`develop`**,
point it at a small web server on your network (the **App-Sync** server), and the
TV pulls the app down over your Wi-Fi. You run that server once, install, then
**close it** — the TV streams Twitch on its own.

> Older **2011–2012** (D/E) sets aren't supported — their MAPLE engine and aged
> TLS make them unreliable.

## Why port 80

On F/H firmware the TV's IP field takes an **IP address only — no port** — so the
TV always fetches `http://<ip>/widgetlist.xml` on **port 80**. The installer
**defaults to port 80** for exactly this reason, so you just start it normally
(macOS asks for your password once, since binding port 80 requires it).

## Option A — with the bundled installer

1. **Get the installer** for your computer (each link is always the latest release):
   - macOS (Apple Silicon, M1–M4): [**twellie-orsay-host-macos-arm64.zip**](https://github.com/nikitakatchik/smarttv-twitch/releases/latest/download/twellie-orsay-host-macos-arm64.zip)
   - macOS (Intel): [**twellie-orsay-host-macos-x64.zip**](https://github.com/nikitakatchik/smarttv-twitch/releases/latest/download/twellie-orsay-host-macos-x64.zip)
   - Windows (64-bit): [**twellie-orsay-host-windows-x64.zip**](https://github.com/nikitakatchik/smarttv-twitch/releases/latest/download/twellie-orsay-host-windows-x64.zip)

   Unzip it (self-contained — it bundles a signed copy of Node, nothing to install)
   and start it:
   - **macOS:** double-click **`Install-Twellie.command`** (first time: right-click
     → **Open** → **Open**), then enter your **Mac password** when asked.
   - **Windows:** double-click **`Install-Twellie.bat`**. If something already uses
     port 80 (e.g. IIS), stop it first, or pass another port: `node.exe host.js 8081`.

   It prints your computer's LAN IP. Leave it running.

2. **Put the TV in `develop` mode and sync** — the menu path **differs by year**:

   ### F-series (2013)
   1. **Menu → Smart Features → Samsung Account → Log In.** Account name: **`develop`**
      (type the literal word — it's a username, **not** your email). The password
      field is usually **greyed out — just confirm**. If yours is typable, try
      `sso1029dev!`, then `000000`, then leave it blank.
   2. From Smart Hub open **More Apps** (bottom) → **Options** (top-right) →
      **IP Setting** → enter the IP the installer printed → confirm.
   3. **Options → Start App Sync.** Wait for **“Service installed.”**

   ### H-series (2014)
   1. **Menu → Smart Hub → Samsung Account → Log In.** Account name: **`develop`**;
      password is **any value** (e.g. `000000`; blank works on some sets).
   2. **Press and hold OK/Enter (~5 s) on any installed app** to open its context
      menu → **IP Setting** → enter the IP the installer printed → confirm.
   3. Hold **OK** again → **Start User App Sync.** Wait for **“Service installed.”**

3. **Open Twellie** and watch. Once it's installed you can **close the installer** —
   the TV streams Twitch directly.

## Option B — without our host (run your own App-Sync server)

The TV's App-Sync client is just an HTTP client; the bundled host only automates
what any web server can do. To do it by hand:

1. **Serve two files from a web server on port 80** on the same LAN:
   - `widgetlist.xml` at the **web root** (`http://<your-ip>/widgetlist.xml`):
     ```xml
     <?xml version="1.0" encoding="UTF-8"?>
     <rsp stat="ok">
       <list>
         <widget id="Twellie">
           <title>Twellie</title>
           <compression size="<ZIP-SIZE-IN-BYTES>" type="zip"/>
           <description></description>
           <download>http://<your-ip>/Twellie.zip</download>
         </widget>
       </list>
     </rsp>
     ```
     `size` must equal the zip's **exact byte count**, and the `<download>` URL is
     followed **verbatim** — point it wherever you actually host the zip.
   - The **widget zip** at that `<download>` URL. Use **`twellie-orsay-unsigned.zip`**
     from the [latest release](https://github.com/nikitakatchik/smarttv-twitch/releases/latest),
     or build it yourself: `npm run build:orsay` then zip the contents of `dist/orsay/`.
2. Any static server works — `sudo python3 -m http.server 80`, Apache/XAMPP
   (`widgetlist.xml` in the web root, zip alongside), nginx, etc. Verify in a
   desktop browser that `http://<your-ip>/widgetlist.xml` returns the XML before
   syncing.
3. On the TV, do the **same `develop`-login + IP Setting + Start App Sync** steps
   from Option A (F or H as above), pointing the IP at your server.

> **USB sideload?** Not recommended on Orsay. The `userwidget` USB folder is a
> **Tizen** (2015+) convention; Orsay's USB path uses an app-named folder, is
> firmware-dependent, and is disabled on most F/H sets. The network App-Sync above
> is the reliable method.

## Troubleshooting

- **Nothing happens on *Start App Sync* (or the TV reboots mid-sync)** — almost
  always the server isn't reachable: it must be on **port 80**, your host
  firewall must allow it, and the TV and computer must be on the same network.
- **The `develop` login fails** — it's the literal username `develop`, not your
  email. On H, any password works; on F the field is usually greyed out (just
  confirm).
- **macOS blocks the launcher** — the first time, right-click
  `Install-Twellie.command` → **Open** → **Open** (the launcher isn't signed). It
  clears the quarantine flag, then asks for your password to bind port 80.
- **A "Commercial break" plays first** — anonymous playback gets Twitch's
  pre-roll ad; it clears to the live stream after a few seconds.
- **Playback stutters** — lower the quality in the player with CH▲ / CH▼.

> Most F/H sets reach Twitch directly. If browsing or playback fails right after a
> clean install, your panel's firmware likely can't negotiate modern TLS to
> Twitch — those sets aren't supported, and there's no fallback (the app connects
> directly; nothing proxies it).
